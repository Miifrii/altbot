# 🎫 Миграция конфигурации панели тикетов

## ✅ Что изменилось

Конфигурация панели тикетов теперь хранится в базе данных вместо хардкода в `config.py`.

### **Преимущества:**
- ✅ Каждый сервер имеет свою конфигурацию панели
- ✅ Настройки сохраняются между перезапусками
- ✅ Редактирование через `/embed_edit` сохраняется в БД
- ✅ Автоматическая миграция из старого конфига
- ✅ Полная обратная совместимость

## 🔄 Как работает миграция

### **Автоматическая миграция:**
При первом обращении к панели тикетов система автоматически:
1. Проверяет наличие конфига в БД для сервера
2. Если конфига нет - импортирует значения из `TICKET_CONFIG["panel"]`
3. Сохраняет в БД
4. Дальше использует только БД

**Пользователь не заметит разницы!**

### **Fallback система:**
```
1. БД (основной источник)
   ↓ если пусто
2. TICKET_CONFIG["panel"] (старый конфиг)
   ↓ если пусто
3. Дефолтные значения
```

## 📋 Что хранится в БД

**Таблица `ticket_panel_config`:**
- `title` - заголовок панели
- `description` - описание панели
- `footer` - текст в footer
- `color` - цвет embed (integer)
- `banner_url` - URL баннера
- `panel_channel_id` - ID канала панели
- `panel_message_id` - ID сообщения панели
- `updated_by` - кто обновил
- `updated_at` - когда обновлено

## 🎮 Команды

### **Существующие команды (без изменений):**

```bash
/panel          # Отправить панель тикетов
/embed_edit     # Редактировать панель
```

**Изменения в поведении:**
- `/embed_edit` теперь сохраняет в БД вместо `TICKET_CONFIG`
- `/panel` сохраняет ID канала и сообщения в БД

### **Измененные файлы:**

- `_get_default_panel_config()` - дефолты

**core.py** (~30 строк изменений):
- `build_panel_embeds(guild_id)` - загрузка из БД
- `PanelEmbedModal.__init__(guild_id, config)` - принимает конфиг
- `PanelEmbedModal.on_submit()` - сохранение в БД
- `send_panel()` - сохранение ID в БД

### **Кэширование:**
- Конфиг панели кэшируется на 5 минут
- Кэш очищается при обновлении конфига
- Каждый сервер имеет свой кэш
