# Альтушка — Discord бот для StarHorizon [18+]

Бот для управления сервером: тикеты, отзывы, роли и утилиты.

---

## Установка и запуск

1. Установи зависимости:
```bash
pip install -r requirements.txt
```

2. Создай файл `.env` в папке `discord-bot/` (скопируй из `.env.example`):
```env
DISCORD_TOKEN=твой_токен_бота
PREFIX=!
```

3. Настрой конфигурацию в `config.py`:
   - Обновить ID серверов в `guild_ids`
   - Настроить ID каналов в `channels`
   - Настроить ID ролей в `roles`
   - Настроить ID категорий в `categories`

4. Запусти бота:
```bash
python bot.py
```

## Команды

### 🎫 Тикеты

| Команда | Права | Описание |
|---|---|---|
| `/panel` | Администратор | Отправить панель тикетов в канал |
| `/embed_edit` | Администратор | Редактировать embed панели тикетов |
| `/ticket_add @user` | Модератор | Добавить участника в тикет |
| `/ticket_remove @user` | Модератор | Убрать участника из тикета |

**Типы тикетов:**
- 🚨 **Жалоба** — пожаловаться на нарушителя
- ⚖️ **Обжалование** — обжаловать наказание
- 🕐 **Перенос времени** — запросить перенос наигранных часов
- 🔞 **Верификация возраста** — пройти верификацию
- 📝 **Другое** — любой другой вопрос

**Кнопки внутри тикета:**
- `Взять тикет` — модератор берёт тикет в работу
- `Передать тикет` — передать другому модератору
- `Закрыть тикет` — закрыть с указанием причины (доступно автору и модераторам)

При закрытии тикета:
- Сохраняется транскрипт переписки в лог-канал
- Автор получает уведомление в ЛС

---

### 📝 Отзывы

| Команда | Права | Описание |
|---|---|---|
| `/review_panel` | Администратор | Отправить панель отзывов в канал |

**Типы отзывов:**
- 🎉 **На ивент** — дата, оценка ивента, оценка ивентолога, текст
- 🛡️ **На администратора** — на кого, причина
- 💜 **Благодарность** — кому, за что
- 📝 **Другое** — на что, текст

**Особенности:**
- Можно отправить анонимно или с указанием автора
- Для ивентов, администрации и другого — выбор тональности (позитивный / негативный / нейтральный)
- Цвет embed'а зависит от тональности
- Кулдаун 60 секунд между отзывами (хранится в БД)
- Поддержка длинных текстов до 4000 символов

---

### 🔐 Роли

| Команда | Права | Описание |
|---|---|---|
| `/roles @user` | Настраивается в БД | Открыть панель управления ролями |

Панель позволяет выдавать и снимать роли в рамках своих прав.
Все действия логируются в лог-канал.
Система ролей и отделов настраивается автоматически из `config.py`.

## Конфигурация

### Основная конфигурация — `config.py`

Все ID каналов, ролей и настройки теперь централизованы в одном файле:

```python
CONFIG = {
    "guild_ids": [123456789, 987654321],  # ID серверов
    
    "channels": {
        "log": 123456789,           # Канал логов
        "reviews": 123456789,       # Канал отзывов
        "reviews_panel": 123456789, # Канал панели отзывов
        "tickets_panel": 123456789, # Канал панели тикетов
    },
    
    "roles": {
        "superadmin": 123456789,    # Суперадмин
        "admin_dept": 123456789,    # Роль отдела администрации
        "moderator": 123456789,     # Модераторы
        # ... остальные роли
    },
    
    "categories": {
        "complaints": 123456789,    # Категория жалоб
        "appeals": 123456789,       # Категория обжалований
        "general": 123456789,       # Общая категория
    },
    
    "tickets": {
        "cooldown_seconds": 20,
        "one_active_per_user": True,
    },
    
    "reviews": {
        "cooldown_seconds": 60,
    },
}
```

## База данных

Бот использует **только SQLite** для хранения всех данных:

- **tickets** — информация о тикетах
- **users** — статистика пользователей
- **ticket_counters** — счетчики тикетов по типам
- **review_counters** — счетчики отзывов по типам
- **user_cooldowns** — кулдауны пользователей
- **departments** — отделы и роли
- **department_roles** — роли отделов
- **role_permissions** — права на выдачу ролей

Миграция из старых JSON файлов происходит автоматически при запуске.

### ✅ Что улучшено:

1. **Единая база данных SQLite** — убраны JSON файлы
