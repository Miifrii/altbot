feat: автоматическая синхронизация тикетов при запуске бота

Проблема:
- После перезапуска бота тикеты показывали "Тикет #0" без типа и автора
- Данные тикетов терялись если бот перезапускался
- Приходилось вручную восстанавливать информацию

Решение:
- Добавлена функция sync_existing_tickets() при запуске бота
- Бот автоматически сканирует категорию тикетов
- Находит каналы с именами "ticket-XXXX" или "тикет-XXXX"
- Проверяет есть ли тикет в БД
- Если нет - добавляет с определением владельца и типа

Как работает:
1. При запуске бот проходит по всем каналам в категории тикетов
2. Извлекает ID тикета из имени канала
3. Определяет владельца по permissions канала
4. Определяет тип по топику канала (жалоба/вопрос/предложение)
5. Добавляет тикет в БД если его там нет

Логирование:
- "[TICKETS] Начинается синхронизация тикетов..."
- "[TICKETS] Синхронизирован тикет #123 (канал: ticket-123, владелец: 456)"
- "[TICKETS] Синхронизация завершена: добавлено X, пропущено Y"

Преимущества:
- ✅ Автоматическое восстановление данных после перезапуска
- ✅ Нет потери информации о тикетах
- ✅ Корректное отображение номера, типа и автора
- ✅ Работает для существующих тикетов

Изменения:
- discord-bot/cogs/tickets/loader.py: добавлена sync_existing_tickets()
